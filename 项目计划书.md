# MMPPPT - 思维导图式PPT展示系统 项目计划书

## 一、项目概述

### 1.1 项目名称
MMPPPT (MindMap PowerPoint)

### 1.2 项目目标
开发一个本地运行的前端应用，将传统PPT的线性展示方式转换为树状思维导图结构，用户可通过点击思维导图节点查看对应的图片内容。

### 1.3 核心特性
- 以思维导图形式展示内容结构
- 点击节点跳转到对应的图片组
- 支持单张或多张连续图片展示
- 纯前端实现，本地运行，无需后端服务
- 树状结构，支持多层级嵌套

## 二、技术栈方案

### 2.1 推荐技术栈
- **前端框架**: Vue 3 + Vite（轻量快速）
- **思维导图库**: 
  - 方案A: Markmap（可直接解析Markdown生成思维导图）
  - 方案B: Mermaid（支持Markdown语法）
  - 方案C: jsMind（纯JavaScript，灵活度高）
- **图片查看**: 
  - 自定义轮播组件
  - 或使用 Swiper.js
- **Markdown解析**: marked.js（如需解析MD文件）
- **样式**: CSS3 / Tailwind CSS

### 2.2 替代方案
- **纯静态**: HTML + CSS + JavaScript（无构建工具）
- **React方案**: React + react-mindmap

## 三、功能需求详细设计

### 3.1 思维导图展示
**输入**: index.md 文件
```markdown
# 项目总览
## 第一章：介绍
### 1.1 背景
### 1.2 目标
## 第二章：方案
### 2.1 技术选型
### 2.2 架构设计
```

**功能**:
- 解析Markdown文件生成思维导图
- 节点支持收缩/展开
- 节点样式可自定义（颜色、大小）
- 支持缩放、拖拽导航

### 3.2 图片展示模块
**功能**:
- 点击节点打开图片浏览器
- 支持单张图片展示
- 支持多张图片的前后翻页
- 键盘快捷键支持（←→翻页，ESC退出）
- 显示当前进度（如：3/5）
- 支持全屏模式

### 3.3 数据配置
**方案一：纯Markdown方式**
```markdown
# 项目总览 [images/cover.png]
## 第一章 [images/ch1/1.png, images/ch1/2.png]
### 1.1 背景 [images/ch1/bg.png]
```

**方案二：Markdown + JSON配置**
- index.md: 定义思维导图结构
- config.json: 定义节点与图片的映射关系
```json
{
  "nodes": {
    "第一章": {
      "images": ["images/ch1/1.png", "images/ch1/2.png"]
    },
    "1.1 背景": {
      "images": ["images/ch1/bg.png"]
    }
  }
}
```

## 六、技术难点与解决方案

### 6.1 Markdown到思维导图的转换
**难点**: 如何将Markdown标题层级转换为树状结构
**方案**: 使用正则表达式解析标题级别（#数量），构建树形数据结构

### 6.2 节点与图片的关联
**难点**: 如何优雅地建立节点和图片的映射关系
**方案**: 
- 简单场景：在Markdown标题后用`[]`标注图片路径
- 复杂场景：使用独立的JSON配置文件

### 6.3 图片预加载
**难点**: 提升用户体验，避免切换时等待加载
**方案**: 点击节点后预加载该节点的所有图片，以及相邻节点的图片

### 6.4 大量图片的性能问题
**难点**: 图片较多时可能影响性能
**方案**: 
- 使用图片懒加载
- 实现虚拟滚动（如果单节点图片超多）
- 图片压缩建议

## 七、用户体验设计

### 7.1 交互流程
1. 打开应用 → 展示思维导图
2. 浏览思维导图结构
3. 点击感兴趣的节点
4. 弹出图片查看器，展示对应图片
5. 通过箭头或键盘翻页查看多张图片
6. 关闭图片查看器，返回思维导图

### 7.2 视觉设计
- 思维导图：中心节点明显，层级清晰
- 图片查看器：暗色背景，图片居中，控制按钮明显
- 过渡动画：流畅的打开/关闭动画
- 响应式：支持不同屏幕尺寸
