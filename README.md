# MMPPPT - 思维导图式PPT展示系统

一个创新的PPT展示工具，将传统的线性PPT转换为树状思维导图结构，让内容组织更加灵活和直观。

## ✨ 特性

- 🧠 **思维导图展示**: 使用 jsMind 将 Markdown 转换为交互式思维导图
- 🖼️ **全屏图片查看**: 点击节点查看关联的图片内容，真正的全屏显示
- ⌨️ **完整键盘导航**: 支持纯键盘操作，无需鼠标
  - 在思维导图中用左右键浏览所有节点
  - 在图片查看器中用左右键无缝切换图片和节点
  - 左右键完全互逆，体验流畅
- 🎓 **学术演示模式**: 无动画切换，即时响应，专业高效
- 🎨 **精美界面**: 基于 Tailwind CSS 的现代化设计
- 📱 **响应式设计**: 适配不同屏幕尺寸
- 🚀 **纯前端实现**: 无需后端，本地即可运行

## 🛠️ 技术栈

- **框架**: Vue 3 + Vite
- **样式**: Tailwind CSS
- **思维导图**: jsMind
- **图标**: Heroicons (SVG)

## 📖 使用说明

### 1. 创建内容结构

编辑 `public/index.md` 文件，使用 Markdown 标题定义你的内容结构：

```markdown
# 我的项目

## 第一章
### 1.1 介绍
### 1.2 背景

## 第二章
### 2.1 方案
### 2.2 实施
```

### 2. 自动生成配置和图片

修改 `index.md` 后，运行以下命令自动生成配置文件和测试图片：

```bash
# 只生成 config.json 配置文件
npm run gen:config

# 只生成图片文件
npm run gen:images

# 一键生成配置和图片
npm run gen:all
```

**工作流程：**
1. `gen:config` - 自动解析 `index.md`，为每个节点生成配置
2. `gen:images` - 根据配置自动生成对应的测试图片
3. `gen:all` - 一次性完成上述两步

生成后的文件：
- `public/config.json` - 节点与图片的映射配置
- `public/images/slide_*.png` - 自动生成的测试图片

### 3. 替换为真实图片（可选）

将 `public/images/` 目录下的测试图片替换为你的真实PPT截图或设计图片，文件名保持一致即可。

### 4. 启动应用

```bash
npm run dev
```

打开浏览器访问 http://localhost:3000

## 🎮 操作指南

### 思维导图操作
- **点击节点**: 查看关联的图片
- **拖拽**: 移动视图
- **滚轮**: 缩放视图
- **键盘导航**: 
  - **→ (右键)**: 进入子节点 / 下一个兄弟节点 / 父节点的下一个兄弟
  - **← (左键)**: 返回父节点 / 上一个兄弟节点的最深后代
  - **Enter / 空格**: 查看当前选中节点的图片
- **点击按钮**: 
  - "放大": 放大视图
  - "缩小": 缩小视图

### 图片查看器操作（全屏模式）
- **← (左键)**: 
  - 如果当前节点有多张图片，切换到上一张
  - 如果已是第一张图片，自动跳转到上一个节点的最后一张
- **→ (右键)**: 
  - 如果当前节点有多张图片，切换到下一张
  - 如果已是最后一张图片，自动跳转到下一个节点的第一张
- **ESC**: 退出图片查看器，返回思维导图
- **点击圆点**: 快速跳转到指定图片（多图时显示）
- **点击背景**: 关闭查看器

### 💡 使用技巧
- 完全支持键盘演示，无需鼠标
- 左右键是完全互逆的操作，可以自由前后浏览
- 图片以全屏方式显示，适合学术演示和会议报告
- 切换无动画，响应即时，更专业

## 🎨 自定义配置

### 修改主题颜色

编辑 `tailwind.config.js`:

```js
theme: {
  extend: {
    colors: {
      primary: '#3b82f6',    // 主色调
      secondary: '#8b5cf6',  // 次色调
    }
  }
}
```

### 修改思维导图样式

编辑 `src/components/MindMap.vue` 中的 jsMind 配置：

```js
const options = {
  container: jsmindContainer.value,
  theme: 'primary',      // 主题：primary, warning, danger, success, info, greensea
  editable: false,       // 是否可编辑
  depth: 4,             // 默认展开层级
  view: {
    hmargin: 120,       // 水平边距
    vmargin: 20,        // 垂直边距
    line_width: 2,      // 连线宽度
    line_color: '#558'  // 连线颜色
  }
}
```

## 💡 使用技巧

1. **组织内容**: 使用清晰的标题层级，建议3-4层最佳
2. **图片命名**: 使用有意义的文件名，便于管理
3. **图片大小**: 建议图片尺寸为 1920x1080 或 1280x800，以获得最佳全屏显示效果
4. **节点标题**: 保持节点标题简洁明了，方便快速识别
5. **图片格式**: 支持 PNG、JPG、SVG 等常见格式
6. **键盘演示**: 
   - 在思维导图中用 → 键依次进入各个节点
   - 按 Enter 进入图片模式
   - 在图片模式用 ← → 键浏览所有内容
   - 按 ESC 返回思维导图选择其他分支
7. **学术演示**: 适合会议报告、课程讲解、项目汇报等正式场合

## 🔧 常见问题

**Q: 图片无法显示？**
A: 检查图片路径是否正确，确保图片在 `public/images/` 目录中。

**Q: 思维导图显示不完整？**
A: 点击"放大"/"缩小"按钮调整视图，或使用鼠标滚轮缩放。

**Q: 如何添加更多层级？**
A: 在 Markdown 中使用更多 `#` 符号表示更深层级（建议不超过4层）。

**Q: 节点点击没有反应？**
A: 确保 `config.json` 中配置了该节点的图片映射，节点名称必须完全匹配。

**Q: 如何进行演示？**
A: 
1. 打开应用后，用 → 键浏览思维导图
2. 按 Enter 进入想要演示的节点
3. 在图片查看器中用 ← → 键浏览所有内容
4. 按 ESC 返回选择其他内容

**Q: 图片没有填满屏幕？**
A: 图片会保持原始比例全屏显示。如果图片比例与屏幕不匹配，会自动适配以完整显示图片内容。

**Q: 键盘导航的逻辑是什么？**
A: 左右键是完全互逆的操作。→ 键按照"深度优先"顺序前进（先子节点，再兄弟节点，最后向上），← 键完全反向操作。
